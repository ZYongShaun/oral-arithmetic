# Proposal: 简化家庭局域网的认证系统

## 概述

将现有的需要用户名+密码的认证系统简化为仅需用户名的快速登录，适用于家庭局域网场景。

## 背景

### 当前问题

- 用户需要记住用户名和密码，对于家庭场景过于复杂
- JWT token 只有2小时有效期，需要频繁重新登录
- 多个孩子共用平板时，切换账户不方便

### 使用场景

- **环境**：家庭 WiFi 局域网，信任度高
- **设备**：平板电脑等（相对固定）
- **用户**：家里的多个孩子轮流使用同一设备
- **需求**：
  - 快速身份切换（孩子A → 孩子B）
  - 持久登录状态（下次打开不用重新输入）
  - 管理员可见：谁在答题（姓名标识）
  - 用户名可以是真实姓名、昵称或混合（不需要区分）

## 目标

### 主要目标

1. **快速登录**：用户只需输入姓名就能登录，无需密码
2. **自动创建用户**：如果用户名不存在，自动创建新用户
3. **长期有效token**：JWT 有效期从 2 小时延长到 2 年
4. **完整管理后台**：家长可以查看所有孩子的进度和统计

### 次要目标

1. **记住登录状态**：用户可以选择是否记住登录状态
2. **最近使用记录**：显示该设备最近使用的用户列表
3. **向后兼容**：旧用户可以继续使用原有的密码登录方式

## 范围

### 包含的内容

1. **后端 API**
   - 新增 `/api/v1/auth/quick-login` 快速登录接口
   - 新增 `/api/v1/users/recent` 获取最近用户接口
   - 调整 JWT 过期时间为 2 年
   - 自动创建用户的逻辑

2. **前端用户端**
   - 修改登录页面（移除密码输入框）
   - 添加"记住登录状态"选项
   - 显示"最近使用用户"列表
   - 优化登录流程

3. **管理后台**
   - 今日数据统计（活跃用户、答题总数等）
   - 本周排行榜
   - 单个用户历史趋势
   - 用户管理（列表、搜索、禁用等）

4. **数据库**
   - User 表保持不变（password_hash 可以为 null）
   - 添加辅助索引用于快速查询

### 不包含的内容

1. 设备管理系统（用户中心设计，不需要设备追踪）
2. 多因素认证（家庭局域网不需要）
3. 复杂的权限系统
4. 短信/邮箱验证

## 方案

### 登录流程

**首次使用（无历史记录）**
```
1. 用户打开 App
2. 输入姓名（如"小明"）
3. 选择"记住我的登录状态"（可选，默认勾选）
4. 点击"开始答题"
5. 后端检查用户是否存在
   - 如果不存在 → 自动创建新用户
6. 生成长期有效的 JWT token（2年）
7. 保存到 localStorage
8. 进入主页
```

**第二次及以后（有历史记录）**
```
1. 用户打开 App
2. 显示最近使用用户列表（如：小明、小红）
3. 用户点击头像或输入新姓名
4. 如果选择了历史用户 → 直接登录
5. 如果是新用户 → 按首次使用流程
```

### 关键决策

| 决策点 | 选择 | 原因 |
|--------|------|------|
| JWT有效期 | 2年 | 方便用户，仍然保留撤销能力 |
| password_hash保留 | 是 | 向后兼容，保持数据库结构一致 |
| 用户名冲突处理 | 前端提示 | 让家长知道并手动处理更有意义 |
| 设备追踪 | 否 | 用户中心设计，简化系统 |

## 成功标准

1. 用户只需输入姓名即可登录
2. 自动创建新用户流程顺畅
3. JWT token 在 2 年内有效
4. 家长可以在管理后台查看所有孩子的进度
5. 旧用户（有密码）可以继续使用原有登录方式
6. 登录页面显示最近使用用户列表

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 用户名冲突 | 用户混淆 | 前端提示"用户名已存在，请添加后缀" |
| 数据迁移问题 | 旧用户无法登录 | 保留 password_hash 字段但可空，老登录方式保留 |
| 局域网外暴露 | 安全问题 | 在部署文档中说明使用场景限制，配置防火墙规则 |
| 多孩子使用同一设备 | 数据混乱 | 用户中心设计，每个用户独立数据 |

## 时间线

### 阶段 3: 管理后台统计功能（优先级中）
- 今日数据统计
- 本周排行榜

### 阶段 4: 测试与优化（优先级低）
- 功能测试
- 性能测试
- UI/UX 优化

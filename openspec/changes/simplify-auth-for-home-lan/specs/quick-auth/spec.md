# Spec: Quick Authentication

## 新增需求

### 需求：系统必须支持仅使用用户名的快速登录
系统必须提供快速登录接口，允许用户仅需提供用户名即可完成身份验证，无需输入密码。

#### 场景：使用快速登录访问系统
- **当** 用户在登录页面仅提供用户名（无密码）并提交
- **那么** 系统必须验证用户名不存在密码字段的要求
- **并且** 系统必须检查用户名是否已存在于数据库中

---

### 需求：系统必须自动创建不存在的快速登录用户
当用户使用快速登录功能提供不存在的用户名时，系统必须自动创建新用户账户。

#### 场景：新用户首次快速登录
- **当** 用户提供的用户名在数据库中不存在
- **那么** 系统必须创建新用户记录，用户名为提供的值
- **并且** 系统必须将用户状态设置为启用（status=1）
- **并且** 系统必须将 password_hash 字段设置为 NULL
- **并且** 系统必须设置创建时间戳为当前时间

#### 场景：已有用户快速登录
- **当** 用户提供的用户名在数据库中已存在
- **那么** 系统不得创建新用户记录
- **并且** 系统必须使用现有用户记录进行身份验证

---

### 需求：快速登录接口必须生成长期有效的 JWT token
系统必须为成功快速登录生成的 JWT token 设置长期有效期（至少1年以上）。

#### 场景：快速登录生成 long-lived token
- **当** 用户通过快速登录成功验证
- **那么** 系统必须生成 JWT token
- **并且** token 过期时间（exp）必须设置为当前时间加上至少1年（建议2年）
- **并且** token 必须包含用户ID作为 sub claim

#### 场景：长期 token 保持有效
- **当** 用户使用快速登录生成的 token 在6个月后访问受保护资源
- **那么** 系统必须验证该 token 仍然有效
- **并且** 系统不得返回 401 Unauthorized 错误

---

### 需求：快速登录接口必须返回用户基本信息
系统必须包括用户身份信息在快速登录响应中，以便前端存储和显示。

#### 场景：快速登录响应包含用户信息
- **当** 用户通过快速登录成功验证
- **那么** 系统必须返回包含以下字段的响应：
  - `access_token`: JWT token 字符串
  - `token_type`: "bearer"
  - `user.id`: 用户整数ID
  - `user.username`: 用户名字符串
  - `user.total_questions`: 该用户总答题数（整数）

---

### 需求：系统必须提供获取最近使用用户的接口
系统必须提供接口允许前端获取最近使用用户列表，用于在登录页显示快捷选项。

#### 场景：获取最近使用用户列表
- **当** 前端请求最近使用用户接口，并提供有效的 JWT token
- **那么** 系统必须验证 token 有效性
- **并且** 系统必须返回用户列表数组 `recent_users`
- **并且** 每个用户对象必须包含：
  - `id`: 用户ID
  - `username`: 用户名
  - `total_questions`: 总答题数

#### 场景：获取最近用户接口未授权
- **当** 前端请求最近使用用户接口，但未提供有效 JWT token
- **那么** 系统必须返回 401 Unauthorized 状态码
- **并且** 系统不得返回用户列表

---

### 需求：系统必须验证用户名的有效性
系统必须在快速登录时验证用户名符合格式要求。

#### 场景：验证用户名长度
- **当** 用户提供的用户名长度小于2字符或大于20字符
- **那么** 系统必须拒绝快速登录请求
- **并且** 系统必须返回 400 Bad Request 状态码
- **并且** 系统必须在响应中包含错误详情"姓名长度必须在2-20个字符之间"

#### 场景：验证用户名格式
- **当** 用户提供的用户名包含非法字符（非中文、英文、数字、下划线）
- **那么** 系统必须拒绝快速登录请求
- **并且** 系统必须返回 400 Bad Request 状态码
- **并且** 系统必须在响应中包含错误详情"只能输入中文、英文、数字和下划线"

---

### 需求：系统必须处理快速登录时的用户名冲突
系统必须正确处理用户名已存在但被禁用的情况。

#### 场景：快速登录用户已被禁用
- **当** 用户提供已存在但 status=0（禁用）的用户名
- **那么** 系统必须拒绝快速登录请求
- **并且** 系统必须返回 400 Bad Request 状态码
- **并且** 系统必须在响应中包含错误详情"User account is disabled"（或中文对应）

---

## 修改需求

### 需求：前端登录页面必须支持快速登录表单
前端登录页面必须提供只包含用户名输入的快速登录选项，移除密码字段要求。

#### 场景：显示快速登录表单
- **当** 用户访问登录页面
- **那么** 页面必须显示一个包含"用户名"字段的表单
- **并且** 页面不得显示"密码"字段
- **并且** 页面必须提供一个复选框用于选择"记住登录状态"（默认选中）
- **并且** 页面必须提供"开始答题"按钮

---

### 需求：前端必须显示最近使用用户列表
前端登录页面必须在有历史记录时显示最近使用用户的快捷选择选项。

#### 场景：显示最近使用用户
- **当** localStorage 中存在 recent_user_ids 且不为空
- **并且** 用户访问登录页面
- **那么** 页面必须在快速登录表单上方显示"选择您的身份"区域
- **并且** 页面必须列出最多5个最近使用的用户卡片
- **并且** 每个用户卡片必须显示：
  - 用户首字头像
  - 用户全名
  - 该用户总答题数

#### 场景：点击最近用户快速登录
- **当** 用户点击最近使用用户列表中的某个用户卡片
- **那么** 前端必须调用快速登录接口，使用该用户名
- **并且** 前端必须在登录成功后保存返回的 token 到 localStorage
- **并且** 前端必须跳转到主页

---

### 需求：前端必须持久化最近使用用户列表
前端必须在每次成功登录后更新 localStorage 中的最近使用用户列表。

#### 场景：更新最近使用用户列表
- **当** 用户成功通过快速登录或选择最近用户登录
- **那么** 前端必须从 localStorage 读取 recent_user_ids 数组
- **并且** 前端必须将当前登录用户的 ID 移动到数组最前面
- **并且** 前端必须移除数组中的重复项
- **并且** 前端必须将数组截断为最多5个元素
- **并且** 前端必须将更新后的数组保存回 localStorage

---

## 移除需求

### 需求：快速登录接口（仅使用用户名）
**Reason**: 需要创建新的专门的快速登录接口。原有的 password-required login 接口保持以向后兼容。
**Migration**: 新的快速登录使用 `POST /api/v1/auth/quick-login`，原有的 password-based登录保持 `POST /api/v1/auth/login` 可用。

---

## 规格说明

### API 端点定义

#### POST /api/v1/auth/quick-login
**描述**: 快速登录接口，仅使用用户名

**请求体**:
```json
{
  "username": "小明"
}
```

**响应 (200 OK)**:
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer",
  "user": {
    "id": 123,
    "username": "小明",
    "total_questions": 99,
    "created_at": "2024-01-15T10:30:00"
  }
}
```

**错误响应 (400 Bad Request)**:
```json
{
  "detail": "用户名已存在但被禁用"
}
```

---

#### GET /api/v1/users/recent
**描述**: 获取最近使用用户列表

**认证**: 需要 Bearer Token

**查询参数**:
- `user_ids` (可选): 用户ID数组，逗号分隔

**响应 (200 OK)**:
```json
{
  "recent_users": [
    {
      "id": 123,
      "username": "小明",
      "total_questions": 99
    },
    {
      "id": 124,
      "username": "小红",
      "total_questions": 35
    }
  ]
}
```

**错误响应 (401 Unauthorized)**:
```json
{
  "detail": "Could not validate credentials"
}
```

---

### 数据存储要求

#### localStorage 数据结构
```javascript
{
  "access_token": "eyJhbGciO...",  // 长期 JWT token
  "user_info": {                    // 当前用户信息
    "id": 123,
    "username": "小明",
    "total_questions": 99
  },
  "recent_user_ids": [123, 124, 125]  // 最近5个使用用户的ID数组
}
```

---

### 前端组件要求

#### Login.vue 必需元素
1. 快速登录表单
   - 用户名输入框（必填，2-20字符，仅中文/英文/数字/下划线）
   - "记住登录状态"复选框（默认选中）
   - "开始答题"按钮

2. 最近使用用户区域（条件渲染）
   - "选择您的身份"标题
   - 用户卡片列表（最多5个）
   - 每个卡片显示：头像、用户名、答题数

---

### 后端数据库要求

#### users 表字段约束
- `username`: VARCHAR(50), UNIQUE, NOT NULL
- `password_hash`: VARCHAR(255), NULLABLE (已修改)
- `status`: INT, DEFAULT 1 (1-启用 0-禁用)
- `created_at`: DATETIME, DEFAULT NOW()
